Traverse Grid

You are given an MxN grid. Each square is either blocked or open.
You are also given K houses, positioned on some squares in the grid (it is guaranteed that each square with a house is open).
Define a route as a sequence of open squares, where each two consecutive squares share a common side.
Your task is to find a route of the shortest length that visits all squares with houses at least once.
You choose the starting, as well as the ending point of the route.

Constraints:: 1 <= M <= 1000
1 <= N <= 1000
1 <= K <= 18

Input: On the first line of the input are the numbers M, N, K (in that order). On each of the next M lines, you are given N symbols, which represent the i-th row of the grid. A ‘.’ means an open square, ‘*’ - closed and ‘H’ means a square with a house.

Output: On the first line of the output, you should output the length of the route you found.
On the second line, you should output the starting row and the starting column you chose.
Note that rows and columns start from 0.
On the third line, you should output a string of directions L, R, U, D representing where your route goes from the current square. L means moving one column to the left, R - one column to the right, U - one row up, D - one row down. Note that the length of the string should match the number you previously output.

Examples: Input: Output: 3 3 2 4 H.. 0 0 … RRDD ..H Input: Output: 4 4 3 6 **H. 1 0 H… RRURDD ***H ....
